Appendix A: Example MVC Implementation Walkthrough
The following is a complete example of how to document a use case using the Model-ViewController pattern. Use this as a reference for Section 1 of your report. Notice that no code
snippets are required -- the goal is to explain what each component does and why, not to paste
your source code into the report.
Use Case: Customer Places an Order
1. The Route
File: routes/orders.js
When the customer submits the checkout form, the browser sends a POST request to
/orders/checkout. This route is protected by a requireAuth middleware that checks whether the
user is logged in. If the user is not authenticated, they are redirected to the login page before
the controller ever runs. If authenticated, the request is handed off to the placeOrder function in
OrderController.
2. The Controller
File: controllers/OrderController.js | Function: placeOrder
The controller is responsible for all business logic. It executes the following steps:
1. Reads the current user's ID from the session.
2. Fetches all CartItem records belonging to that user, including the associated Product for
each item so that the current price is available without a separate query.
3. Calculates the order total by summing quantity multiplied by price for each cart item.
4. Creates a new Order record with the user ID, calculated total, and a status of 'pending'.
5. For each cart item, creates an OrderItem record storing the order ID, product ID,
quantity, and the price at the time of purchase. The price is copied as a snapshot so that
future price changes by the vendor do not affect historical order records.
6. Deletes all CartItem records for the user, clearing their cart.
7. Redirects the user to the order confirmation page for the newly created order.

total DECIMAL Total order value at time of purchase

An Order belongs to one User (customer) and has many OrderItems. The total is stored directly
on the Order so it does not need to be recalculated from line items, which may change over

OrderItem
Field Type Notes
id INTEGER Primary key, auto-increment
orderId INTEGER Foreign key -> Order.id
productId INTEGER Foreign key -> Product.id
quantity INTEGER Number of units purchased
price DECIMAL Unit price at time of purchase (snapshot -- not a live FK

OrderItem is a join table between Order and Product that also carries data. The price field is a
snapshot of the product price at time of purchase. This ensures historical orders stay accurate
even if the vendor changes the price later.
